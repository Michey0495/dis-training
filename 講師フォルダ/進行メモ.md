## 環境トラブルの想定と対処

### Copilotが無反応

- VS Codeを再起動
- Copilotの認証状態を確認（右下のアイコン）
- それでもダメならペアで1台を使ってもらう

### Pythonが入っていない

- VS Codeのターミナルで `python --version` を確認
- python3 で動く場合はエイリアスの問題。手順書のpythonをpython3に読み替え
- 最悪の場合、Python実行は講師のデモを見てもらい、spec.mdとAgent MDの作成に集中

### git initでエラー

- Gitが入っていない → `git --version` で確認
- 「このフォルダでgit initしてください」を明確に伝える
- Git操作は最悪省略してもハンズオンの核心には影響しない

### Agent MDが反映されない

- 既存チャットでは効かないことがある → 新しいチャットを開く
- ファイルパスを確認: `.github/copilot-instructions.md`（ドット付き、サブフォルダなし）
- Ctrl+Shift+P → Reload Window を実行

## TRY IT / ハンズオン 巡回ポイント

### Session 02 TRY IT（文脈なし/あり比較）

見るべきこと:
- Copilot Chatが開けているか（Ctrl+Shift+I）
- #file: の書き方がわかっているか（ファイル名を打ち始めると候補が出る）
- 出力が英語になっている場合 → 「日本語で出力してください」を追加

### Session 02 TRY IT（制約指示）

見るべきこと:
- 制約なしバージョンと制約ありバージョンの出力差に気づいているか
- 差に気づいた受講者がいたら全体共有する

### Session 02 TRY IT（コメントドリブン）

見るべきこと:
- practice.py を作成しているか
- Tab補完が出ているか（出ない場合はEnterで改行して少し待つ）
- コメントだけ書いてAIに補完させる体験ができているか

### Session 02 ハンズオン（仕様書作成）

見るべきこと:
- 「メモに書かれていない要件は追加しないで」の制約を入れているか
- spec.md をファイルとして保存しているか
- git commit まで到達しているか

よくあるつまずき:
- #file: の書き方がわからない → ファイル名を打ち始めると候補が出る
- 出力が長すぎる → 「Markdownテーブル形式で簡潔に」と追加指示
- git commit でエラー → ユーザー名/メールが未設定の場合がある

### Session 03 TRY IT（Agent MDなし/あり比較）

見るべきこと:
- リネーム操作に成功しているか（パスの間違い注意）
- 新しいチャットで試しているか
- 2つの出力の差に気づいているか

よくあるつまずき:
- Windowsで `ren` コマンドの書き方 → VS Codeのエクスプローラーでリネームしてもよい
- 差が出ない → Reload Window を試す

### Session 03 TRY IT（インラインチャット）

見るべきこと:
- Ctrl+I（Cmd+I）でインラインチャットが開けているか
- 関数の選択ができているか（行番号左のクリック、Ctrl+Shift+Kなど）
- 説明・リファクタリング・テスト生成の3操作を試しているか

### Session 03 ハンズオン（Agent MD作成）

見るべきこと:
- .github/ フォルダがプロジェクトルート直下にあるか（サブフォルダに作っていないか）
- copilot-instructions.md がテンプレートのまま放置されていないか（自チーム用に編集しているか）
- 新しいチャットで効果を確認しているか

よくあるつまずき:
- mkdir .github でエラー（Windows） → VS Codeのエクスプローラーから作成
- 効果を確認できない → 新しいチャットを開く + Reload Window
- 何を編集すればいいかわからない → 「自チームの禁止事項を1つ追加する」だけでもOK

### Session 04 TRY IT（バイブコーディング vs 仕様駆動）

見るべきこと:
- バイブコーディングの指示を曖昧なまま出しているか（それが目的）
- 仕様駆動の指示で spec.md と Agent MD を #file: で渡しているか
- 2つの出力の質的な差に気づいているか

このTRY ITは5分と短いので、全員が完了しなくても「差が見えた」時点で進む。

## フォールバック対応

### Session 02 ハンズオン（残り10分で仕様書ができていない人）

spec.md のテンプレートを口頭で共有。最低限の構造だけ作って git commit させる。
全員が「成果物として spec.md を持っている」状態にすることが優先。

### Session 03 ハンズオン（Agent MDが効かない人）

Ctrl+Shift+P → Reload Window で解決することが多い。
それでもダメなら、テンプレートのcopilot-instructions.mdを配置した状態で git commit だけさせる。
